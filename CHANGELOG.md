## [1.13.2] - 2026-06-05

### Fixed
- **연차 패널 연도 이동 중 race condition** — Firebase 콜백 클로저가 `currentYear` 를 참조로 잡아 연도 이동 시 잘못된 연도의 localStorage 키에 저장되던 문제. 콜백에서 연도를 값(`const year`)으로 캡처하도록 수정
- **미동기화 안내 문구** — 폐기된 `npm run sync` 안내를 "확장 팝업에서 로그인 후 동기화" 로 교체
- 확장 팝업 **로그인 실패 메시지가 4초 후 사라지던 문제** — 실패 사유를 다음 시도 전까지 유지하도록 변경

### Changed
- **Chrome 확장(연차 자동 동기화) v1.1.0** — 그룹웨어 SSO에 인증 단계가 추가돼 쿠키(`credentials:include`)만으로 API 인가가 안 되는 문제 대응. `webRequest`로 그룹웨어 페이지가 `jiran.api.groupware.pro` 에 실제로 보내는 인증 헤더(`Authorization` 등)를 가로채 저장하고, 동기화 호출 시 함께 실어 보냄 → 인증 방식이 쿠키든 토큰이든 자동 적응
- 확장 팝업에 **그룹웨어 연동 상태** 표시 — 토큰 확보됨 / 세션 쿠키로 정상(최근 동기화 성공 시) / 미확보(인사 화면 열기 안내). 401/403 알림도 "토큰 미확보"와 "인증 만료(재로그인)"로 구분
- 동기화 실패 에러에 **실제 HTTP 상태코드 + 응답 본문 일부** 포함 — 원인 파악 용이
- 확장 팝업에 **비밀번호 재설정 이메일 전송** 버튼 추가 (이전 'v1.13.3' 커밋 기능 — 앱이 아닌 확장 기능이라 확장 버전 라인으로 편입)

> 참고: 위 변경 중 캘린더 앱(`index.html`) 자체 수정은 연차 패널 race condition·안내 문구 2건이며, 나머지는 동반 업데이트된 Chrome 확장(v1.1.0)의 변경이다.

- 이미지 복사 시 **상단 툴바 통째로** (이전/다음/오늘/메모/복사/내보내기/불러오기/인쇄/로그인) + **🔒 예정 일정** + **📝 메모 아이콘** + **🔗 URL 아이콘** 등 내부/조작 요소는 캡처에서 자동 제외. 헤더는 한국어 제목 + 영문 부제 둘 다 표시

- 이미지 캡처 시 **디자인 폭 1600px 으로 강제** + **모바일/태블릿 미디어 쿼리를 인라인 스타일로 우회** — 좁은 뷰포트에서 토요일 컬럼이 잘리거나 제목이 2줄로 줄바꿈되던 문제 해결. 항상 데스크탑 레이아웃으로 캡처
- 사내 게시판용 복사에서 완료/최종 뱃지를 **인라인 텍스트 라벨**로 변경 — 게시판이 `display:inline-table` 을 무시하여 큰 블록 박스로 깨지던 문제 해결

- 캘린더 기본 폰트를 **맑은 고딕(Malgun Gothic) 우선**으로 변경 — Windows 환경에서 일관된 한글 폰트 렌더. Mac에서는 Apple SD Gothic Neo로 폴백
- 캘린더 헤더를 **수직 스택** (제목 위 + 툴바 아래) 으로 재구성 — 11개 버튼이 제목을 밀어 줄바꿈하던 문제 해결. 툴바는 `flex-wrap: wrap` 으로 좁은 화면에서 자연스럽게 다음 행으로 떨어짐

## [1.6.2] - 2026-05-13

### Changed
- URL 자동 전파 시 **"재확인" 마커를 무시** — 다음이 모두 동일한 일정으로 취급되어 URL이 함께 등록/수정/삭제됨:
  - `회의 X` ↔ `회의 X 재확인`
  - `회의 X` ↔ `회의 X (재확인)` / `[재확인]` / `「재확인」`
  - `회의 X` ↔ `회의 X - 재확인` / `: 재확인` / `, 재확인`
  - `회의 X` ↔ `회의 X+재확인1차` / `회의 X+재확인2차` (`+`/`-` 결합 + N차 카운터)
  - `회의 X 2차` ↔ `회의 X 2차+재확인1차` (다른 카운터와 결합된 경우)
- URL 자동 전파가 **다른 월의 일정에도 적용** — 한 달에서 URL 을 등록/수정/삭제하면 모든 월에 걸쳐 같은 base 일정에 일괄 반영됨. Firebase 의 `/calendar` 트리를 한 번 읽고 영향받는 월만 다중 경로 업데이트(`db.ref().update({...})`)로 일괄 쓰기
- 새 일정 추가 시 같은 base 의 기존 일정에 URL 이 있으면 자동 상속 — 현재 월에 없으면 다른 월에서도 비동기로 검색하여 발견 시 새 일정에 URL 을 채워 넣고 재렌더

### Fixed
- 일정이 많은 날짜의 모달을 열 때 **헤더(우측 상단 [전체 삭제]/[×] 버튼)가 화면 위쪽으로 밀려 안 보이던 문제** — 모달 `max-height` 를 92vh 로 제한하고 본문 영역(`.modal-body`)만 내부 스크롤되도록 변경 (flex column 레이아웃)

## [1.6.1] - 2026-05-12

### Changed
- 한 일정에 URL 을 등록/변경하면 **같은 텍스트의 다른 모든 일정**에 같은 URL 이 자동 적용됨. 예: "회의 X" 일정이 5일/12일/19일에 있을 때 한 곳만 등록하면 나머지에도 즉시 반영. URL 을 비우면 같은 텍스트 일정들의 URL 도 같이 제거되어 "동일 텍스트 ⇒ 동일 URL" 불변식 유지
- 캘린더 타일의 🔗 아이콘을 **마우스 오버가 아닌 상시 표시**로 변경. [완료]/[최종] 뱃지 바로 위에 작은 탭 형태로 매달림. **태그 높이는 그대로 유지**되고 텍스트 폭도 줄지 않음

## [1.6.0] - 2026-05-12

- `/annual_leave/_meta.lastSyncSource` 필드 — 어느 경로(`"extension"`/스크래퍼)에서 마지막으로 갱신됐는지 식별
